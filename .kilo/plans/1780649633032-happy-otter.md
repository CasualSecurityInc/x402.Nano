# Remove `tagAmountRaw` from Track A (`nanoTxn`)

## Background

`tagAmountRaw` was introduced as a unique-amount fingerprint to prevent cross-challenge borrowing. The actual attack scenario it addresses (MITM + cross-facilitator frontrunning with matching base amounts) is:

1. **Nano-specific** — EVM EIP-3009 permits bind to a spender/facilitator address, so cross-facilitator replay is cryptographically impossible there. Nano state blocks lack a `spender` field, making them facilitator-agnostic.
2. **Out of scope for x402** — the x402 protocol assumes MITM-free HTTP transport. Cross-facilitator replay is not a concern the payment mechanism needs to solve.
3. **DoS, not theft** — even if exploited, the victim just gets `STALE_FRONTIER` and reconstructs.
4. **Not in the canonical reference** — x402nano/schemes/exact.md uses plain fixed amounts with no fingerprinting.

## Decisions

- **Expiry**: Single field — `extra.validBefore`, Unix timestamp in seconds. This is the canonical x402 v2 field name used in the core spec, all v2 transports (HTTP, MCP, A2A), and the EVM exact scheme. Track B already uses this name. Drop both the old ISO-8601 `expiresAt` and the dual `maxTimeoutSeconds`+`expiresAt` redundancy.
- **`extra` field**: Kept, containing only `expiresAt`. Not empty, not removed.
- **MITM**: Accepted as out of scope per x402 transport assumptions.

## Plan

### 1. Remove `tagAmountRaw` from PaymentRequirements

- Remove `extra.tagAmountRaw` field from the `PaymentRequirements` JSON example and table
- Replace `extra.expiresAt` (ISO-8601 string) with `extra.validBefore` (Unix timestamp integer, seconds — matching the canonical x402 v2 field name)
- Remove the "Amount Tagging" subsection entirely
- `accepted.amount` is now a plain fixed amount with no tag offset
- Remove `maxTimeoutSeconds` or repurpose it as purely informational (not a second expiry)

### 2. Update PaymentPayload

- Remove `extra.tagAmountRaw` from the `accepted` field in the Payload example
- Rename `extra.expiresAt` → `extra.validBefore`, change format to Unix timestamp integer

### 3. Simplify Facilitator Verification

- Remove `extra.tagAmountRaw` format check (step 1)
- Replace ISO-8601 `expiresAt` check (step 2) with Unix timestamp check: `extra.validBefore <= floor(now_unix_seconds)`
